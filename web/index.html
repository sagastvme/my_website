My Philosophy

This site avoids fancy animations and heavy frameworks. I believe in simplicity, efficiency, and minimal dependencies. This approach keeps the site fast, light, and accessible, allowing me to focus on core tasks.

Using HTML’s natural semantic structure makes content accessible to all. Complex animations add unnecessary complexity, so I focus on simplicity—not to avoid problem-solving, but to prioritize business logic and productivity.

This Website is Basic, and That’s How I Like It

This website is straightforward—no frills, just what I need. Some people may prefer complexity, but I enjoy simplicity. It might not impress everyone, but it serves my purpose perfectly.

The site is hosted on Lighttpd in a Docker container on an Oracle VPS, with JavaScript for reactive data. I also used ChatGPT to confirm correct HTML tags for accessibility. A reverse proxy enables HTTPS access, and NoIP provides a free domain.

I prefer updating via SSH to live-update the code (although not recommended) over maintaining a fancy portfolio that may be prone to breakage or lack responsiveness.

Monolithic or Serverless

It depends. For a small company with a primary need for a CRM, I’d suggest a monolithic structure, perhaps with a cloud-hosted database to manage sensitive data securely.

Serverless options may be overkill for simpler needs. Know what’s best for your use case, and embrace simplicity when it’s effective.

When faced with lagging performance, consider hybrid scaling and moving heavy compute tasks to serverless solutions, focusing on Infrastructure as Code for long-term benefits.

Typed Languages or No Types

I now lean towards typed languages, though I still appreciate Python for its simplicity. Typed languages help maintain standards and improve code quality.

Choosing a JavaScript Framework

Can you accomplish your goals with vanilla JavaScript? Replicating framework features with vanilla JS is highly educational, as I’ve learned from experience.

SPA or Server-Side Website?

SPAs are interesting, but browsers already offer routing, caching, etc. Use SPAs if they add real value, but remember you have other options.

JSON or HTML Responses?

If there’s no need for a mobile app, return HTML for simplicity. Alternatively, consider an API that supports a ‘/json’ mode for flexibility.

Learning Git is Essential

Version control is crucial. Familiarize yourself with tools like Jenkins, Git CLI, and open-source contributions—it’s rewarding and vital for growth.

Master Your IDE

Learn all shortcuts and tools your IDE offers; it will significantly improve your productivity.

Take Care of Your Posture and Ergonomics

Understanding posture, ergonomic tools, and potential risks of being a developer is essential. Learn about carpal tunnel, kyphosis, and myopia risks, and remember to stretch and give your eyes breaks regularly.

Use AI Sparingly

AI is helpful for specific tasks and debugging, but rely on it sparingly. Avoid AI dependency when you can achieve results manually, as it’s often more reliable and efficient.
